Instellingen webservices

In dit dialoogvenster (Algemene configuratie > Webservices) kan je volgende opties instellen:

  • Wachtwoord: Dit is het globaal wachtwoord dat toegang geeft tot alle methodes (string $accesscode). Je kan het wachtwoord tijdelijk zichtbaar maken door op het oogje naast het wachtwoord te klikken. Via het kopieer-icoon kan je het wachtwoord naar je klembord kopiëren.
  • Doelgroep: De groep waarin de nieuw aangemaakte gebruikers terechtkomen.
  • Uniek veld: Kies gebruikersnaam of intern nummer. Afhankelijk van deze keuze zal Smartschool inkomende requests behandelen (string $userIdentifier).

Het globaal wachtwoord verleent aan een externe toepassing automatisch toegang tot alle methodes. Dat is zeker niet altijd gewenst! Maak bij voorkeur een profiel aan met een apart wachtwoord voor beperkte toegang tot methodes.

Globaal wachtwoord

  1. Kies in 'Algemene configuratie' voor 'Webservices'.
  2. Bekijk of wijzig het globaal wachtwoord bij 'wachtwoord' en klik eventueel op Opslaan.

Profielen met beperkte toegang

  1. Kies in 'Algemene configuratie' voor 'Webservices'.
  2. Klik op 'Profiel toevoegen' om een nieuw profiel aan te maken.
  3. Vul de naam van het profiel in en duidt aan tot welke methoden dit profiel toegang heeft.
  4. Klik op Opslaan.

Bezorg het wachtwoord van dit profiel aan de persoon die de externe toepassing beheert. Met dit wachtwoord heeft deze toepassing enkel toegang tot de door jou gekozen methodes.

Webservices V3 (SOAP)

Het is mogelijk om zelf aan de slag te gaan met de webservices van Smartschool. Hieronder vind je alle informatie van de verschillende methoden terug.

  • https://school.smartschool.be/Webservices/V3?wsdl

Voorbeelden van toepassingen in PHP en C#

We bieden enkele concrete voorbeelden aan van programmacode in C# en PHP waarbij de webservices en OAuth gebruikt worden:

  • C#-toepassing om het wachtwoord en de status van een gebruiker te wijzigen.
  • PHP-toepassing om het wachtwoord van een gebruiker te wijzigen.
  • PHP-toepassing voor het bestellen van broodjes, authenticatie met behulp van OAuth.

De broncode van alle toepassingen kan je downloaden ter inspiratie. Het is niet de bedoeling dat je deze toepassingen effectief gebruikt. Ze zijn bedoeld als aanzet en voorbeeld.

Detail van de methoden

addCourse

Met deze methode is het mogelijk om een vakken aan te maken in Smartschool. Het XML-bestand heeft 3 parameters aan boord:

  • string $accesscode: Wachtwoord webservices
  • string $coursename: Vaknaam
  • string $coursedesc: Vakomschrijving
  • int $visibility: Zichtbaarheid (optioneel: '1' = zichtbaar (default) of '0' = niet zichtbaar)

addCourseStudents

Met deze methode is het mogelijk om bepaalde klassen toe te voegen aan een bepaald vak. Het XML-bestand heeft 4 parameters aan boord:

  • string $accesscode: Wachtwoord webservices
  • string $coursename: Vaknaam
  • string $coursedesc: Vakomschrijving
  • string $groupIds: CSV-lijst met unieke klas- of groepscode(s)

addCourseTeacher

Met deze methode is het mogelijk om een vakleerkracht en co-leerkrachten toe te voegen aan een bepaald vak. Het XML-bestand heeft 5 parameters aan boord:

  • string $accesscode: Wachtwoord webservices
  • string $coursename: Vaknaam
  • string $coursedesc: Vakomschrijving
  • string $internnummer: Uniek veld gebruiker
  • string $userlist: CSV-lijst met uniek veld gebruiker

changeGroupOwners

Met deze methode is het mogelijk om de klastitularissen van een klas te wijzigen.

  • string $accesscode: Wachtwoord webservices
  • string $code: Unieke klas- of groepscode
  • string $userlist: CSV-lijst met uniek veld gebruiker

changeInternNumber

  • string $accesscode: Wachtwoord webservices
  • string $username: Gebruikersnaam
  • string $newInternNumber: Nieuw intern nummer

changeUsername

Met deze methode is het mogelijk om de gebruikersnaam van een gebruiker te wijzigen waarbij het intern nummer als sleutelveld fungeert.

  • string $accesscode: Wachtwoord webservices

  • string $newUsername: Nieuwe gebruikersnaam

checkStatus

Met deze methode kan de status van de synchronisatieopdracht van de methode startSkoreSync opgevraagd worden.

  • string $accesscode: Wachtwoord webservices
  • string $uuid: De unieke code van de synchronisatieopdracht van de methode startSkoreSync

Deze methode geeft de status van de synchronisatieopdracht terug:

  • no_status: er is voor de opgegeven UUID geen opdracht gevonden
  • busy: de opdracht is bezig
  • done: de opdracht is voltooid
  • error: de opdracht is niet voltooid door een fout tijdens de opdracht
clearGroup

Deze methode verwijdert alle gebruikers uit een groep.

  • string $accesscode: Wachtwoord webservices

saveClass

Met deze methode is het enerzijds mogelijk om klassen en officiële klassen aan te maken of te wijzigen in Smartschool. Het is evenwel niet mogelijk om de klasnaam van bestaande klassen en officiële klassen te wijzigen. Het xml-bestand heeft 6 parameters en 3 optionele parameters aan boord.

Indien er nog geen (officiële) klas voorkomt met de aangeleverde identificatie string ($name) zal Smartschool een nieuwe (officiële) klas aanmaken. Bestaat er echter wel reeds een (officiële) klas met de identificatie string dan zullen de bestaande parameters in Smartschool worden aangepast.

  • string $parent: Unieke klas-of groepscode van parent groep (leeg of '0' voor hoogste niveau)
  • string $untis: Koppelingsveld schoolagenda
  • string $instituteNumber: Instellingsnummer (optioneel: enkel voor het aanmaken van een officiële klas)
  • string $adminNumber: Nummer van de administratieve groep (optioneel: enkel voor het aanmaken van een officiële klas)
  • string $schoolYearDate: Dit past het instellingsnummer en nummer van de administratieve groep aan voor het doorgegeven schooljaar (indien niet meegegeven wordt dit aangepast voor het huidige schooljaar)

saveUser

Voegt een gebruiker toe of wijzigt een bestaande gebruiker.

Als de saveUser methode wordt aangeroepen dan wordt het ingesteld uniek veld (userIdentifier) gecontroleerd. Indien er nog geen account bestaat met het ingesteld uniek veld dan zal er een nieuwe account worden toegevoegd. Bestaat er echter wel reeds een account met het ingesteld uniek veld dan worden de gegevens van deze account geüpdatet met de meegeleverde gegevens.

  • string $accesscode: Wachtwoord webservices
  • string $internnumber: Uniek veld gebruiker
  • string $username: Gebruikersnaam (minstens 2 karakters)
  • string $passwd1: Wachtwoord hoofdaccount
    Voor nieuwe gebruikers is het verplicht om dit wachtwoord op te geven. Bij bestaande gebruikers is het mogelijk om deze string leeg te laten, zo zal het wachtwoord van deze gebruiker niet gewijzigd worden.
    Lees hier alle informatie over de voorwaarden van een wachtwoord of wachtzin.
  • string $passwd2: Wachtwoord co-account (optioneel)
  • string $passwd3: Wachtwoord tweede co-account (optioneel)
  • string $name: Voornaam (minstens 1 karakter)
  • string $surname: Achternaam (minstens 1 karakter)

Wanneer het adres in de vorm <straatnaam> spatie <huisnummer> spatie 'bus' spatie <busnummer> wordt aangeboden zal de saveUser service dit herkennen de gegevens in de juiste databankvelden invoeren. Bv.: Kerkstraat 22 of Kerkstraat 22 bus 1. Naast het woord 'bus' wordt ook een schuine streep gebruikt om huisnummer van busnummer te scheiden. Kerkstraat 22/1 wordt dus Kerkstraat 22 bus 1.

saveUserToClass

Met deze methode verplaats je de gebruiker naar de (nieuwe) officiële klas. Een leerling kan maar aan één officiële klas gekoppeld worden. De officiele datum kan optioneel worden meegeleverd.

  • string $accesscode: Wachtwoord webservices
  • string $userIdentifier: Uniek veld gebruiker
  • string $class: Unieke klas- of groepscode
  • string $officialDate: Officiële datum (optioneel: zonder officiële datum dient de klasbeweging in Smartschool te worden bevestigd; formaat: YYYY-MM-DD)

saveUserToClasses

Met deze methode kan een gebruiker aan meerdere klassen/groepen gekoppeld worden. De nieuwe klassen en groepen worden aan de bestaande klassen en groepen toegevoegd. In geval van een officiële klas: de leerling zal verwijderd worden uit zijn huidige officiële klas en opnieuw gekoppeld worden aan de officiële klas die wordt meegegeven. De methode vereist 3 parameters:

  • string $accesscode: Wachtwoord webservices
  • string $userIdentifier: Uniek veld gebruiker
  • string $csvList: CSV-lijst met unieke klas- of groepscode(s)

saveUserToClassesAndGroups

Via deze methode is het mogelijk om de lidmaatschappen van de officiële klas, klassen en groepen van een bepaalde gebruiker te bepalen. Het is mogelijk om de nieuwe lidmaatschappen toe te voegen aan de bestaande of om de bestaande lidmaatschappen eerst te verwijderen. Indien er fouten zijn bij de verwerking dan wordt een CSV-bestand met de foutcodes teruggegeven.

  • string $accesscode: Wachtwoord Webservices
  • string $userIdentifier: Uniek veld gebruiker
  • string $csvList: CSV-lijst met unieke klas- of groepscode(s)
  • integer $keepOld: ('1' = behoud bestaande lidmaatschappen en voeg nieuwe lidmaatschappen toe, '0' = verwijder oude lidmaatschappen en maak enkel de nieuwe lidmaatschappen)

sendMsg

Via deze methode kan je een bericht naar de hoofdaccount of een co-account van een bepaalde gebruiker sturen. Het opgeven van de verzender en/of bijlage is optioneel.

  • string $accesscode: Wachtwoord webservices
  • string $userIdentifier: Uniek veld gebruiker van de ontvanger
  • string $title: Titel van het bericht
  • string $body: Tekst van het bericht
  • string $senderIdentifier: Uniek veld gebruiker van de verzender
  • mixed $attachments: Bijlagen (1 of meer) in base64 encoding (optioneel)
  • integer $coaccount: Type account ('0' = hoofdaccount, '1' = co-account 1 of '2' = co-account 2 …)
    $coaccount is optioneel: indien dit niet wordt opgegeven dan wordt het bericht naar de hoofdaccount gestuurd.
  • boolean $copyToLVS: Waarde 'true' voegt het bericht toe aan het LVS (optioneel en standaardwaarde is 'false')

De array van de bijlagen ziet er in pseudo code als volgt uit:

[ ['filename' ⇒ 'test1.docx', 'filedata' ⇒ 'base64encoded string'] ['filename' ⇒ 'test2.docx', 'filedata' ⇒ 'base64encoded string'] ]

Of je kan een JSON-string meegeven met de volgende structuur:

[
{ “filename”: “test1.docx”, “filedata”: “base64encoded string” },
{ “filename”: “test2.docx”, “filedata”: “base64encoded string” }
]

setAccountPhoto

Met deze methode kan de foto van de leerling ingevuld worden. Door een 'lege' foto door te sturen kan de foto ook gewist worden. De functie accepteert 3 parameters:

  • string $accesscode: Wachtwoord webservices
  • string $userIdentifier: Uniek veld gebruiker
  • string $photo: base64encoded photo

setAccountStatus

Met deze methode kan de status van een leerling gewijzigd worden. De methode vereist 3 parameters:

  • string $accesscode: Wachtwoord webservices
  • string $userIdentifier: Uniek veld gebruiker
  • mixed $accountStatus: Status account

Deze laatste parameter kan volgende waarden hebben:

  • 'actief', 'active' of 'enabled' om de account op actief te zetten
  • 'inactief', 'inactive' of 'disabled' om de account uit te schakelen
  • 'administrative' of 'administratief' om van de account een administratieve account te maken.

startSkoreSync

Met deze methode kan de synchronisatie van klassen en leerlingen van Smartschool naar Skore gestart worden. Deze methode levert een JSON-object met daarin de UUID van de synchronisatieopdracht.

  • string $accesscode: Wachtwoord webservices

Deze opdracht kan slechts één keer per 15 minuten gestart worden.
